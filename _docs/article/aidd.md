**项目名称：AI端到端全流程托管交付**
**项目时间**：2025.3 ~2025.9
**项目介绍**：构建AI端到端需求交付平台，用户在Web端录入需求并开启托管交付，由需求Agent拆解为结构化开发与测试任务；编码Agent调用RAG增强检索、代码编辑、代码编译等工具链自动生成可运行代码；测试Agent创建并执行多维度自动化测试（接口/UI/回归），输出完整报告。通过需求→编码→测试阶段多智能体协同，实现从需求到可验证代码的全自动交付闭环。

**项目功能：**

* 自然语言需求自动拆解：用户在需求管理平台录入需求，开启托管交付流程，由需求Agent自动解析为结构化开发任务与测试任务，并分发至编码Agent与测试Agent。
* 自主式端到端代码生成：编码Agent基于开发任务，自主调用文件读取、语义检索、代码编辑，编译诊断等工具，在无需人工干预下完成技术实现方案和代码生成。
* 二轮精准交互式修正：首轮输出技术实现方案和代码文件后；支持用户对具体文件提出修改意见，实现局部纠偏修正。
* 提供实时编译反馈能力：编译过程实时反馈（含错误/警告及颜色标识）至前端，用户可即时验证生成代码的正确性。
* 自动化测试闭环执行：在代码通过编译并成功部署后，测试Agent依据测试任务自动创建测试用例，执行多维度自动化测试(接口/UI/回归)，生成完整的测试报告。

**技术栈：**Spring AI、LangEngine、OpenSearch、Tis3Plus、JavaParser
**职责描述：**核心主力开发，负责编码Agent的开发，包括并不限于：

* 设计并实现基于ReAct范式的自主式编码Agent执行引擎
  * 基于LangEngine框架定制回调生命周期，构建LLM与工具链的闭环交互机制，构建支持状态感知与迭代控制的ReAct执行循环。
  * 设计并实现一套面向代码生成场景的高内聚工具集，包括：文件读取、内容检索、代码修改等三大类共10个原子工具。
  * 制定“强自主式”Prompt策略，弱化协作，明确要求Agent主动推进直至完成任务。
  * 难点：多步工具链及工具的设计；LLM无故终止流程提前退出；无人工干预的Agent执行。
  * 效果：无故提前终止率0%，任务完成率提升至95%。
* 主导构建中英双通道混合检索通道

  * 构建 中文语义通道（LLM生成的注释 + 中文关键词）与 英文结构通道（基于CAST论文的算法分块代码+英文关键词），分别适配自然语言需求理解与代码结构检索。

  * 构建项目级领域词典库，统一中英术语表达，指导 Query 改写。

  * 差异化粗召回策略，抑制 “关键字霸权”，平衡 “高精度” 与 “高召回”。

  * 精排引入自适应聚类对多路粗召回结果分组后，结合LLM驱动的精排打分机制，融合功能正确性（CodeBERT语义匹配 + 动词对齐）与代码质量（标识符F1匹配），实现高精度的需求-代码重排序。

  *  难点：高频词霸权；结构同质误召；同义异构漏召；精排协同评估功能正确性与代码质量。

  * 效果：召回Recall@20达90%。精排显著提升排序区分度—将高度相似的粗排分数（如0.8），重排为层次分明的分数（如0.2/0.5/0.9），精准识别与需求强相关的高质量代码片段。
* 构建分层代码编辑Apply体系，显著提升大文件修改成功率与速度。
  * 针对不同文件规模动态选择edit_file(小文件)或search_replace(大文件)策略;
  * 集成 Qwen2.5 Fast Apply模型加速小文件编辑，并设计 “Fast Apply→大模型→Search/Replace” 三级回退机制，确保修改的可靠性与落地成功率。
  * 难点：原始全量重写方式在大文件上apply速度慢（>1min）、易出错、token容易达到上限；Search/Replace在重复代码段存在替换歧义，因空格/格式敏感导致匹配失败。
  * 效果：小文件apply成功率达100%，平均耗时降至20s内；大文件apply成功率98%平均耗时1s内。
* 实现多级代码校验体系，保障生成代码可编译、可运行

  * 构建 “单文件AST语法校验 + 多JDK版本Maven全项目编译” 双层诊断体系，覆盖语法错误与依赖/版本兼容性问题。
* 通过日志监听 + SSE流式推送实现可视化实时编译过程反馈。
  * 难点：支持JDK/Lombok多版本动态适配Maven编译环境；在无阻塞前提下实现低延迟、可靠的编译过程流式反馈。
* 效果：语法校验平均编译耗时低于1秒，成功率100%。Maven编译耗时处于分钟级，可检测依赖冲突等问题，成功率≥95%，并兼容菜鸟线上主流JDK版本，编译过程流式反馈，最大观测延迟控制在1秒内。

> ## CAST: Code-Aware Semantic Tree
>
> CAST（Code-Aware Semantic Tree）是一篇关于代码表示学习的研究论文，提出一种面向语义的树状结构来捕捉程序的语义信息。该方法旨在弥合源代码的语法结构与语义理解之间的差距，在代码分析、生成及理解任务中取得更优表现。
>
> ### 关键事实
>
> - **研究主题**：代码语义表示与程序理解
> - **核心方法**：语义感知的树结构（Semantic Tree）
> - **应用场景**：代码搜索、代码摘要、自动补全
> - **提出机构/作者**：通常由程序语言与人工智能领域研究团队提出
> - **主要贡献年份**：2020年代初期
>
> ### 背景与动机
>
> 传统的代码表示方法（如AST，抽象语法树）主要基于语法层面，难以全面表达程序的语义。CAST提出在保留语法信息的同时，引入语义节点和关系，使模型能更好理解代码的功能行为。这为代码智能处理任务提供了更深层次的语义基础。
>
> ### 方法与架构
>
> CAST通过构建“语义增强树”结构，将变量依赖、函数调用、控制流等语义关系显式编码。模型在图神经网络或树结构网络的支持下，对节点和边进行信息传播，从而学习代码的上下文语义。
>
> 4
>
> ### 应用与影响
>
> 该方法被用于多个代码智能任务，如代码相似度计算、漏洞检测与代码生成。实验表明，相较于仅基于语法的模型，CAST在理解复杂代码语义、减少误匹配方面更具优势。
>
> ### 当前发展
>
> CAST的思路推动了代码语义建模的进一步研究，催生了结合大型语言模型（如CodeBERT、GraphCodeBERT）的混合方法。其语义树思想仍被广泛引用，用于提升代码理解与生成系统的精确性与可解释性。



> ## JavaParser
>
> JavaParser 是一个用 Java 编写的开源语法分析与代码操作库，可将 Java 源代码解析为抽象语法树（AST），并支持代码的分析、转换与生成。它广泛用于构建编译器前端、静态分析工具和自动化重构程序，是最受欢迎的 Java 代码解析框架之一。
>
> ### 关键事实
>
> - **初始发布**：2008 年
> - **主要维护者**：Danny van Bruggen（又名 Matozoid）
> - **语言支持**：Java 1.0–21
> - **许可协议**：Apache 2.0 或 LGPL 双许可证
> - **核心组件**：JavaParser Core、JavaSymbolSolver、AST 序列化模块
>
> ### 工作原理
>
> JavaParser 读取 Java 源文件并生成 AST（抽象语法树），开发者可通过 API 访问和修改节点，从而执行语法分析、代码检测或自动重构。
> 其扩展组件 **JavaSymbolSolver** 提供符号解析功能，能识别变量、方法和类型声明之间的关系，用于类型推断、引用查找等高级分析任务。
>
> ### 功能与应用
>
> 该库可：
>
> - **分析**：遍历 AST 寻找指定的代码模式；
> - **转换**：修改现有语法节点以实现重构；
> - **生成**：自动创建 Java 类、方法等结构；
> - **序列化**：将 AST 导出为 JSON 以供其他工具使用。
>   JavaParser 常被用于 IDE 插件、代码质量工具（如 SonarQube 插件）及科研项目。
>
> ### 社区与生态
>
> 项目由活跃的全球社区维护，拥有超过两百名贡献者及数千个依赖项目。其官方站点 [javaparser.org](https://javaparser.org/) 提供免费教程与电子书《*JavaParser: Visited*》，示例工程包括 Maven 与 Gradle 快速入门模板。
